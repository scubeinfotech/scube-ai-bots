📱 WhatsApp Support Guide

🎯 Quick Reference for Support Staff

Webhook URL Pattern:

/api/whatsapp/webhook/{tenant_id}

Default Verify Token:

scube_wa_verify_2024

🔄 Two-Way Configuration Flow

1

What Tenant GETS from MSG91:

  • Auth Key (e.g., 516065A2X1NLCv6a1167dcP1)
  • Integrated Number (e.g., 6580786788)
  • API Endpoint (usually default)

→ Tenant enters these in their dashboard

2

What Tenant GIVES to MSG91:

  • Webhook URL (auto-generated per tenant)
  • Verify Token (scube_wa_verify_2024)

→ Tenant copies these from dashboard to MSG91

🛠️ Common Issues & Solutions

"Messages not received by customer"

→ Check config_metadata has correct msg91_auth_key

→ Verify integrated_number matches MSG91 dashboard

→ Ensure + prefix in phone numbers (our code auto-adds this)

"Webhook verification fails"

→ Verify token must be exactly: scube_wa_verify_2024

→ Webhook URL must include correct tenant_id

→ Check tenant has active WhatsApp config in database

"Health check error"

→ Database schema issue - run fix_whatsapp_schema.py

🔍 Database Verification Query

SELECT tenant_id, phone_number_id, webhook_verify_token,
config_metadata->>'msg91_auth_key' as auth_key,
config_metadata->>'msg91_integrated_number' as integrated
FROM whatsapp_configurations
WHERE tenant_id = 'TENANT_ID_HERE';
